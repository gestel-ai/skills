# Meta And Instagram Static Learning Rules

This reference supports static product-photo Creative Packages. Platform delivery systems and benchmarks change, so verify live sources before making dated or external performance claims.

## What To Learn From Static Variants

Compare variants by:

- Template family.
- Layout ID.
- Ratio.
- Angle.
- Claim pattern.
- Visual pattern.
- Product-photo treatment.
- CTA.
- Badge or offer type.
- Review Gate warning or failure category.

## First Recommendations

Prefer changes with real conceptual difference:

- Different template family over minor color swaps.
- Different claim pattern over synonym-only headline edits.
- Different visual composition over small background changes.
- Review-proof or offer-led tests only when evidence exists.

## Fatigue And Weak-Signal Notes

Use caution when:

- Impressions or clicks are too low for directional confidence.
- Frequency rises while CTR falls.
- Spend is uneven across variants.
- Date ranges differ.
- Post-click metrics conflict with CTR.
- A Review Gate warning may explain poor performance.

## Next-Test Hypothesis Shape

```text
Because [observed pattern from mapped variants],
we believe [next template family / angle / slot change]
will improve [primary metric]
for [audience or placement],
while monitoring [guardrail metric].
```

## Recommendation Boundaries

- Recommend the next Creative Package or launch draft fields, not live campaign changes.
- Do not infer audience targeting or budget changes unless the CSV includes enough context and the user asks for analysis.
- Do not use competitor or platform trend claims as confirmed-current without live verification or a dated source.
